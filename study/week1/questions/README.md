## 기정

**Q1. BST 말고 레드-블랙 트리를 쓰는 이유?**
> 재정렬할 때 성능이 더 좋다. BST는 편향될 수 있어 최악 $O(n)$이지만, 레드-블랙 트리는 $O(\log n)$ 보장.

**Q2. 이진 탐색을 하기 위한 전제조건?**
> 오름차순 또는 내림차순으로 **정렬**되어 있어야 한다.

**Q3. 정렬 안 된 데이터에서 1번만 데이터를 찾는다면 어떤 게 효율적일까?**
> **선형 탐색**이 더 효율적. 정렬 자체에 시간이 걸리기 때문에, 여러 번 접근하는 경우에만 정렬 비용이 이득이 된다.

**Q4. 여러 번 탐색을 한다면?**
> $O(n)$을 여러 번 하면 $O(n^2)$. 정렬하면 $O(n \log n)$ + 이후 탐색마다 $O(\log n)$ → 탐색 횟수가 많을수록 유리하다.

**Q5. 재귀가 몇천만 번까지 일어난다면?**
> 함수 호출이 많아지면 스택 영역을 사용 → 힙 영역을 침범 → **Stack Overflow** 발생

**Q6. 캐시 히트, 캐시 미스 경험이 있으신지?**
> 하드웨어가 빨라서 직접 경험하긴 어렵지만, Kingfisher로 이미지 처리할 때 관련됨. 컴퓨터 구조 공부할 때 메모리 그리면서 이해했음.

**Q7. Swift에서 함수가 stack 영역인가? 함수 자체가 stack 영역인가?**
> 내가 알기로는 지역변수나 그런 것들이 스택 메모리에 들어가는걸로 알고 있는데.. 정확하게는 모르겠음 찾아봐야할듯.
