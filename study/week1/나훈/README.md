## Hash Table

- 키를 해시 함수로 인덱스로 바꿔서 빠른 조회를 할 수 있게 함
- Dictionary, Set이 Hash Table 기반
- 키를 해시 함수로 넣음 → 해시 값 생성 → 배열의 인덱스처럼 활용

**시간복잡도**

| 연산 | 평균 | 최악 |
|------|------|------|
| 삽입 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |
| 탐색 | O(1) | O(n) |

- 최악의 경우: 해시 충돌이 발생했을 때 체이닝(Chaining)을 통해 처리
  - 실제로는 키 값이 중복되는 경우가 적어 평균 O(1)에 수렴

**Dictionary vs Set**

| 자료구조 | 구성 |
|---------|------|
| Set | 키만 가지고 있음 |
| Dictionary | 키-Value 쌍 |

## Tree

- 부모와 자식 간의 관계를 계층적으로 표현하는 자료구조
- 비선형 자료구조
- 계층 구조가 자연스러운 경우에 사용
  - 파일 시스템, 메뉴 구조, 댓글의 답글과 같은 상하관계

**단순 Tree의 탐색**: O(n)

→ 이를 개선한 게 **BST (이진 탐색 트리)**: 평균 O(log n)
- 데이터가 나보다 작으면 왼쪽, 크면 오른쪽

→ BST를 더 개선한 게 **Red-Black Tree**
- 작은 거 왼쪽, 큰 거 오른쪽 규칙은 동일
- 작은 것만 계속 들어온다면? Red / Black 단계로 나눠서 균형 유지
- 최악의 경우에도 O(log n) 보장

## Heap / Priority Queue

- 단순히 먼저 들어온 걸 처리하는 게 아니라, **작업 중요도 / 최단 거리 / 스케줄링** 같은 걸 처리할 때 Priority Queue 사용

**시간복잡도**
- 데이터 삽입 / 삭제: **O(log n)**

**동작 방식**
- 내가 먼저 넣은 데이터라도 우선순위가 밀리면 내부적으로 재정렬됨
- 결론적으로 항상 우선순위대로 정렬된 상태 유지
- 꺼낸 순서가 우선순위대로 나오기 때문에 정렬 목적으로도 활용 가능

**Swift에서 Priority Queue**
- 표준 라이브러리에 없어서 직접 구현해야 함 (약 100줄...)
- 배열(완전 이진 트리 구조)로 구현
- 우선순위 변경 시 부모와 자식 위치를 인덱스로 교환 → 완전 이진 트리이기 때문에 인덱스 계산이 유리

## Queue

- **FIFO (First In First Out)** — 선입 선출
- 먼저 들어온 데이터가 먼저 나가는 구조

| 연산 | 시간복잡도 |
|------|-----------|
| enqueue (삽입) | O(1) |
| dequeue (삭제) | 평균 O(1) |

- `removeFirst()` 사용 시 내부적으로 모든 요소를 앞으로 이동 → O(n) 주의
- **BFS(너비 우선 탐색)** 에서 주로 활용
